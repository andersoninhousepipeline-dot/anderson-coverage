F8 Gene Coverage Analysis

Gene: F8 (Coagulation Factor VIII)  |  Locus: chrX:154,835,787–155,026,940 (GRCh38, minus strand)  |  Span: 191.2 kb  |  Transcript: ENST00000360256 / NM_000132.4 (MANE Select, Ensembl canonical)  |  Exons: 26  |  Total exonic bases: 9,032 bp

Clinical relevance: Pathogenic variants in F8 cause Haemophilia A (X-linked recessive). The gene spans ~186 kb on the X chromosome with 26 exons. Intron 22 inversion accounts for ~45% of severe cases. Complete exonic coverage is essential for comprehensive diagnostic panels.
ⓘ
What this analysis measures

These BED files are capture panel design files — they define which genomic coordinates a probe set is designed to enrich, not what was sequenced in any particular sample. No BAM file is involved.

The coverage figures are computed with bedtools coverage -a f8_exons.bed -b panel_merged.bed and answer: "How many base pairs of each F8 exon fall inside a probe interval in this panel?" This is purely geometric interval overlap.

Panel BED (this analysis)
  • Is F8 designed to be captured?
  • Which exons are targeted and at what breadth?
  • Panel design gaps — variants structurally missed
BAM + depth tools (not done here)
  • Was F8 actually sequenced in a specific sample?
  • Was depth sufficient at each exon (≥20×–30×)?
  • Per-sample QC — library prep failures, low-complexity regions

✓ This approach is correct for panel design assessment. It is not a substitute for per-sample depth QC from a BAM.

●
Additional steps required for full clinical validation
  1. Per-sample depth QC (BAM required)
    Run mosdepth --by panel.bed sample.bam or samtools depth -b panel.bed to confirm each F8 exon is covered at ≥20× (minimum) or ≥30× (recommended for reliable heterozygous SNV detection) in every clinical sample.
  2. Known variant hotspot check (ClinVar)
    Cross-reference uncovered exon bases against ClinVar pathogenic/likely-pathogenic F8 variants. A probe gap overlapping a known hotspot is a reportable panel limitation. ROCHE intervals already carry ClinVar allele IDs in their annotation field — these can be extracted directly.
  3. Exon boundary / splice-site padding
    Clinical guidelines (ACMG/AMP, ISO 15189) typically require ±10–20 bp of flanking intronic sequence per exon to capture splice-site variants. Verify whether the panel BED files already include this padding.
  4. Intron 22 inversion — structural variant caveat
    Accounts for ~45% of severe Haemophilia A. This ~600 kb inversion is not detectable by short-read NGS capture panels regardless of probe design. Requires MLPA, inverse-PCR, or long-read sequencing.
  5. Large deletions / duplications (CNV)
    Multi-exon deletions/duplications account for ~5–10% of F8 pathogenic variants. Requires CNV calling from normalised read-depth (GATK gCNV, ExomeDepth, CODEX2) or MLPA — standard SNV/indel pipelines will miss them.

Metric Definitions

All metrics are derived from the five BED files using bedtools v2.30.0 and Python / pandas. Coordinates are 0-based half-open (BED convention). Gene annotations are from Ensembl GRCh38 release 114.

General BED Metrics

Metric Definition Formula / Note
Raw Intervals Total number of rows in the original BED file after stripping header lines (browser / track / # lines). COUNT(all BED rows)
Merged Intervals Number of non-overlapping intervals after running bedtools merge on the sorted BED file. Adjacent or overlapping regions within the same file are collapsed into a single interval. COUNT(bedtools merge output rows)
Raw Bases (bp / Mb) Sum of all interval lengths before merging. May double-count bases if intervals overlap within the same file. SUM(end - start) for all raw intervals
Merged Bases (bp / Mb) Total number of unique base pairs covered after merging overlapping/adjacent intervals. This is the true genomic footprint of the panel. SUM(end - start) for merged intervals
Redundancy (%) Percentage of raw bases that are duplicated due to overlapping intervals within the same file. High redundancy suggests probe overlap or multiple transcripts targeting the same region. ( 1 − Merged Bases / Raw Bases ) × 100
Mean / Median / Q1 / Q3 Interval Size (bp) Descriptive statistics of raw interval lengths. Median and quartiles are more robust to outliers (e.g. very large exon 14 of F8). Q1 = 25th percentile, Q3 = 75th percentile. Computed on the 'size = end − start' column of raw intervals
Min / Max Interval Size (bp) Smallest and largest raw interval in the file. Extremes can indicate probe design artefacts or very large targeted regions (e.g. introns captured in gene panels). MIN/MAX(end − start)
Std Dev (bp) Standard deviation of raw interval sizes. A high value indicates a wide spread in probe sizes, common in whole-exome kits that target both small and large exons. σ(end − start)
Chromosomes Number of distinct chromosomes/contigs referenced by the BED file. Panels targeting specific genes or chromosomal regions will show a lower count. COUNT(DISTINCT chrom)

Pairwise Overlap

Metric Definition Formula / Note
Overlap (% of Row File) Percentage of the row-file's merged bases that are also covered by the column-file. This is directional: row→column may differ from column→row when file sizes differ. Intersect(A ∩ B) / MergedBases(A) × 100
Shared Bases (Mb) Raw count of base pairs present in both files simultaneously after both are individually merged. Computed with bedtools intersect on the two merged BED files. SUM(bp in bedtools intersect -a mergedA -b mergedB)
Shared Interval Segments Number of contiguous merged intervals output by bedtools intersect. A single overlapping region between two panels counts as one segment. COUNT(rows in bedtools intersect -a mergedA -b mergedB)
Jaccard Index Symmetric similarity measure ranging 0 (no overlap) to 1 (identical). Accounts for the total combined footprint of both panels. |A ∩ B| / |A ∪ B| = Shared Bases / (BasesA + BasesB − Shared Bases)

Unique / Shared Regions

Metric Definition Formula / Note
Unique Bases (bp / Mb) Bases covered exclusively by this panel and not by any other panel. Computed with bedtools subtract after unioning all other panels. bedtools subtract -a mergedA -b UNION(all other panels)
Unique (%) Proportion of this panel's merged bases that are not covered by any other panel. High uniqueness indicates panel-specific targets (e.g. disease-specific probes). Unique Bases / Merged Bases × 100
Shared Bases (bp) Bases covered by this panel that are also covered by at least one other panel. Merged Bases − Unique Bases
4-way Intersection Bases covered simultaneously by all four panels. Represents the common core captured regardless of which panel is used. Sequential bedtools intersect across all 4 merged BED files

Size Distribution Bins

Metric Definition Formula / Note
Bin ranges (<50, 50–100 … >10k bp) Raw intervals are assigned to size bins based on their length (end − start). Each cell shows the count and percentage of total intervals in that panel falling into the bin. Useful for assessing probe length uniformity. Histogram of (end − start) values with fixed bin edges

F8 Gene Metrics

Metric Definition Formula / Note
Transcript Reference ENST00000360256 / NM_000132.4 (F8-202). MANE Select and Ensembl canonical transcript. 26 exons, chrX:154,835,788–155,026,940, minus strand (GRCh38/hg38). Source: Ensembl GRCh38.114 GTF. Ensembl ENSG00000185010, HGNC:3546, NCBI Gene:2157
Exonic Coverage (%) Percentage of total canonical exonic bases (summed across all 26 exons) covered by the panel's merged F8 intervals. This is the primary clinical adequacy metric for F8. SUM(bases covered per exon) / SUM(exon sizes) × 100
Gene Span Covered (%) Percentage of the full gene body (from transcription start to end, ~191 kb) covered by the panel's F8 intervals. Usually lower than exonic coverage because introns are not targeted. Merged F8 Bases / Gene Span × 100
Exons Fully Covered Number of exons where 100% of exonic bases are overlapped by at least one panel interval after merging. COUNT(exons where bedtools coverage fraction = 1.0)
Exons Partially Covered Number of exons with >0% but <100% coverage. Partial coverage can indicate probe design gaps or probe boundary effects and may result in missed variants at exon edges. COUNT(exons where 0 < fraction < 1.0)
Exons Not Covered Exons with zero base-pair overlap with any panel interval. Variants in these exons would not be detected by this panel. COUNT(exons where fraction = 0.0)
Mean Exon Coverage (%) Average per-exon coverage percentage across all 26 exons. Differs from overall exonic coverage (%) because it weights each exon equally regardless of its size. MEAN(pct_covered per exon) ≠ Exonic Coverage (%) when exon sizes vary
Per-Exon Coverage (bp) Number of base pairs within a specific exon that are covered by at least one merged panel interval. Computed with bedtools coverage -a exonBED -b panelMergedBED. bedtools coverage column 6 (bases_covered)

Tools & Methods

Metric Definition Formula / Note
bedtools merge Collapses overlapping or book-ended intervals in a sorted BED file into a single interval. Run on each panel individually before computing coverage and overlap statistics. bedtools merge -i sorted_input.bed
bedtools intersect Identifies intervals that overlap between two BED files. Used here on the per-file merged BED files to compute pairwise shared bases and interval counts. bedtools intersect -a mergedA.bed -b mergedB.bed
bedtools subtract Removes intervals in file B from file A. Used to compute bases unique to each panel by subtracting the union of all other panels. bedtools subtract -a mergedA.bed -b union_others.bed
bedtools coverage For each interval in file A (-a), counts how many bases are covered by intervals in file B (-b) and reports the fraction. Used for per-exon F8 coverage. bedtools coverage -a exons.bed -b panel_merged.bed → cols: bases_covered, size, fraction
